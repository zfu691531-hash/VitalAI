# Docs Archive

This directory stores historical design and development materials.

It is intentionally not part of the default reading path for new development sessions. Start from:

1. `docs/DOCS_INDEX.md`
2. `docs/PROJECT_CONTEXT.md`
3. `docs/CURRENT_STATUS.md`
4. `docs/NEXT_TASK.md`
5. `README.md`

## Structure

- `steps/`: historical `STEP_*` implementation notes.
- `design-assets/`: historical docx, mmd, png, and xmind design assets.
- `reports/`: records that are no longer current source-of-truth documents.

Only read archived files when a current task directly depends on the historical decision or detailed design they contain.
