## 三、核心架构约定

### 前后端对称

你的 `src/api/` 下每个文件对应后端一个路由模块：

| 你的文件 | 后端路由 | 基础路径 |
|---|---|---|
| `api/auth.js` | `api/auth.py` | `/api/auth` |
| `api/user.js` | `api/users.py` | `/api/users` |
| `api/student.js` | `api/students.py` | `/api/students` |
| `api/teacher.js` | `api/teachers.py` | `/api/teachers` |
| `api/class_.js` | `api/classes.py` | `/api/classes` |
| `api/score.js` | `api/scores.py` | `/api/scores` |
| `api/schoolRule.js` | `api/school_rules.py` | `/api/school-rules` |
| `api/aiTools.js` | `api/ai_tools.py` | `/api/ai` |
| `api/aiHistory.js` | `api/ai_history.py` | `/api/ai-history` |

### 统一响应格式

所有接口返回：
```json
{ "code": 200, "msg": "操作成功", "data": { ... } }
```
`api/request.js` 的响应拦截器统一处理：`code !== 200` 时 `ElMessage.error(msg)`。

## 四、权限体系（必须遵守）

### 三级角色

| 角色 | 标识 | 数据范围 |
|---|---|---|
| 学生 | `student` | 仅个人数据 |
| 教师 | `teacher` | 本班数据 |
| 校务管理员 | `admin` | 全校数据 |

### 页面级权限

每个路由的 `meta.requiredRoles` 数组声明允许访问的角色，`router/index.js` 全局前置守卫拦截，无权限跳转 `/403`。

### 按钮级权限

使用 `v-permission` 指令：
```vue
<el-button v-permission="['admin']">删除</el-button>
<el-button v-permission="['teacher', 'admin']">编辑</el-button>
```

### 权限矩阵速查

| 页面 | 学生 | 教师 | 管理员 |
|---|---|---|---|
| 学生管理 | 禁 | 本班 | 全量 |
| 教师管理 | 禁 | 仅自己 | 全量 |
| 班级管理 | 禁 | 绑定班级 | 全量 |
| 成绩管理 | 仅自己 | 本班 | 全量 |
| 校规管理 | 禁 | 禁 | 全量 |
| 校规问答 | 可 | 可 | 可 |
| 模拟面试 | 可 | 可 | 可 |
| 其他AI工具 | 禁 | 可 | 可 |
| 分组-教师端 | 禁 | 可(本班) | 禁 |
| 分组-校务端 | 禁 | 禁 | 可 |

## 五、可复用组合式函数（composables）

## 九、代码质量红线

1. **禁止**组件中直接写后端地址，所有请求走 `api/` 层
2. **禁止**组件中直接操作localStorage，走 `stores/user.js`
3. **禁止**重复写表格分页/搜索逻辑，使用 `useTable`
4. **禁止**重复写AI生成loading/retry逻辑，使用 `useAiGenerate`
5. CRUD页面表格列定义、操作按钮位置保持一致
6. 所有表单弹窗统一使用 el-dialog
7. AI生成内容必须添加"仅供参考"提示文字
