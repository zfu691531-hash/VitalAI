## M1：基础配置层（地基，所有模块依赖它）

### 目标
搭建全局配置、统一响应、安全工具，其他模块的运行基础。

### 需要实现的文件（6个）

| 序号 | 文件 | 核心内容 |
|---|---|---|
| 1 | `core/config.py` | `Settings(BaseSettings)` 读取.env，全局单例 `settings` |
| 2 | `core/response.py` | `ResponseModel(BaseModel)` + `success_response()` + `error_response()` |
| 3 | `core/security.py` | `hash_password()` + `verify_password()` + `create_access_token()` + `decode_access_token()` |
| 4 | `database/connection.py` | `engine` + `SessionLocal` + `get_db()` 依赖注入 |
| 5 | `database/base.py` | `Base(DeclarativeBase)` + `TimestampMixin`（created_at + updated_at） |
| 6 | `utils/logger.py` | 日志配置：控制台+文件，按日期轮转 |

### 关键决策

- 配置读取：pydantic-settings 读 `.env`，禁止硬编码

## M3：Schema校验层

### 目标
定义所有接口的Pydantic请求/响应模型，前后端交互的"契约"。

### 需要实现的文件（9个）

| 序号 | 文件 | 核心Schema |
|---|---|---|
| 1 | `schemas/common.py` | `PaginationParams`(page, page_size), `BatchDeleteRequest`(ids), `IdRequest`(id) |
| 2 | `schemas/user.py` | `LoginRequest`(username, password, role), `TokenResponse`(access_token, user_info), `PasswordChangeRequest` |
| 3 | `schemas/student.py` | `StudentCreate`, `StudentUpdate`, `StudentQuery`(keyword, class_id, tag, gender), `StudentResponse` |
| 4 | `schemas/teacher.py` | `TeacherCreate`, `TeacherUpdate`, `TeacherQuery`, `TeacherResponse` |
| 5 | `schemas/class_.py` | `ClassCreate`, `ClassUpdate`, `ClassQuery`(grade, status), `ClassResponse` |
| 6 | `schemas/score.py` | `ScoreCreate`, `ScoreUpdate`, `ScoreQuery`(student_id, class_id, exam_batch, subject), `ScoreResponse`, `ScoreStatsResponse` |
| 7 | `schemas/ai_history.py` | `AiHistoryQuery`(tool_type), `AiHistoryResponse` |
| 8 | `schemas/school_rule.py` | `SchoolRuleCreate`, `SchoolRuleUpdate`, `SchoolRuleQuery`(category), `SchoolRuleResponse` |
| 9 | `schemas/__init__.py` | 统一导出 |

> **注意**：8个AI工具的请求Schema不单独建文件，而是在M11的各`services/ai/*.py`中内联定义Pydantic模型，因为每个AI工具参数差异大且与service逻辑紧耦合。

### 设计原则
- Create schema：必填字段 + 校验规则（如 score 范围 0-100）
- Update schema：所有字段 Optional
- Query schema：继承 PaginationParams，加筛选字段
- Response schema：与前端展示字段对齐，关联字段返回名称而非仅ID

## M6：学生管理CRUD（最复杂，作为模板）

### 目标
完整的学生全生命周期管理，包含所有CRUD、权限隔离、数据一致性校验、批量导入导出。

### 需要实现的文件（3个）

| 序号 | 文件 | 核心接口 |
|---|---|---|
| 1 | `services/student_service.py` | get_list(三级权限过滤), create(校验class_id+student_no唯一), update, delete(校验关联成绩), batch_delete, import_excel, export_excel |
| 2 | `api/students.py` | GET/POST/PUT/DELETE /api/students + /batch-delete + /import + /export + /template |
| 3 | `services/user_service.py` | get_student_by_user_id(user_id) → Student（通过user.name匹配student.name关联） |

### 关键业务逻辑
1. **三级权限隔离**：student→仅自己, teacher→仅本班, admin→全量
   - **student角色**：通过`user.name`匹配`student.name`获取自己的学生记录
   - **teacher角色**：解析`teacher.class_ids`逗号分隔字符串，查询`student.class_id`在该列表中的学生
   - **admin角色**：全量查询无过滤
2. **新增校验**：class_id是否存在、班级是否已满(max_count)、student_no唯一性
3. **删除校验**：是否有关联score记录，有则拒绝
4. **current_count同步**：增/改/删学生时同步更新班级current_count
5. **批量导入**：调用M5的export_service解析Excel→逐行校验→返回成功数+错误明细

### 自测验证
在临时main.py中追加students路由，启动后通过Swagger测试：
- 学生列表按角色过滤正确
- 新增学生到已满班级被拒绝
- 删除有成绩的学生被拒绝
- current_count自动更新
- 批量导入返回错误明细
