## 三、核心架构：三层分离

```
路由层 (api/)         → 接收HTTP请求，参数校验（Pydantic），调用Service，返回统一响应
    ↓
业务逻辑层 (services/) → 核心业务处理：CRUD、权限校验、AI调用、数据一致性检查
    ↓
数据访问层 (database/)  → SQLAlchemy ORM，参数化查询，屏蔽数据库细节
```

**严禁**在路由层写业务逻辑，**严禁**在Service层直接拼SQL。

## 五、认证与权限

### JWT + bcrypt（不是MD5！）

需求文档提到MD5但检查清单明确禁止 → 架构决策采用bcrypt。**不可改回MD5**。

```python
# core/security.py
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")
def hash_password(plain: str) -> str
def verify_password(plain: str, hashed: str) -> bool
def create_access_token(data: dict) -> str
def decode_access_token(token: str) -> dict
```

### 依赖注入（api/deps.py）

```python
get_current_user(token, db)     # 从JWT解析用户，401无权限
require_role(*roles)             # 角色校验，403权限不足
```

路由使用方式：
```python
@router.get("/api/students")
async def list_students(
    db: Session = Depends(get_db),
    user: User = Depends(require_role("teacher", "admin"))
):
    ...
```

### 三级数据隔离（service层实现）

每个查询接口按角色过滤：
- `student` → WHERE id = 当前用户的student_id
- `teacher` → WHERE class_id IN (当前教师的class_ids)
- `admin` → 不过滤

## 六、数据库层

### 7张表

user / student / teacher / class / score / ai_history / school_rule

### 无外键约束

所有关联通过ID字段实现，不设FOREIGN KEY。数据一致性由你在service层保证。

### ORM基类

```python
class Base(DeclarativeBase): pass
class TimestampMixin:
    created_at: Mapped[datetime] = mapped_column(default=datetime.now)
    updated_at: Mapped[datetime] = mapped_column(default=datetime.now, onupdate=datetime.now)
```

所有模型继承Base和TimestampMixin。

### teacher.class_ids 逗号分隔

```
示例值："1,3,5" → 该教师绑定班级ID为1、3、5
```

查询时：Python拆分后传给SQLAlchemy的 `.in_()` 方法。

### class.current_count 同步维护

增删改学生时必须同步更新班级的current_count。

## 七、统一响应格式

所有接口返回：
```json
{ "code": 200, "msg": "操作成功", "data": { ... } }
```

使用 `core/response.py` 的 `success_response()` / `error_response()` 封装。


## 十、数据一致性检查（service层红线）

| 操作 | 必须检查 |
|---|---|
| 删除学生 | 是否有关联成绩？有则拒绝 |
| 删除教师 | 是否为某班班主任？是则拒绝 |
| 删除班级 | 班级内是否有学生/成绩？有则拒绝 |
| 新增学生 | class_id是否存在？班级是否已满？ |
| 批量导入 | 逐行校验关联ID有效性，返回错误明细 |

## 十一、安全红线

1. 密码：bcrypt哈希，**禁止**明文或MD5
2. SQL：SQLAlchemy ORM参数化查询，**禁止** f-string拼接
3. 输入：Pydantic Schema校验，**禁止** request.get('input')
4. 密钥：pydantic-settings读.env，**禁止**硬编码
5. 文件上传：uuid重命名 + 限制扩展名(.xlsx/.xls) + 限制大小
6. 错误信息：统一code/msg/data，**禁止**暴露内部堆栈
