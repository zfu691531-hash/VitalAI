### 第二阶段：认证与权限

```
9. api/deps.py（公共依赖注入，所有路由都依赖它）
   from fastapi import Depends, HTTPException, status
   from jose import JWTError

   async def get_current_user(token: str = Depends(oauth2_scheme), db = Depends(get_db)):
       """从JWT解析当前用户，验证token有效性"""
       try:
           payload = decode_access_token(token)
           user_id = payload.get("sub")
           user = db.query(User).filter(User.id == user_id).first()
           if not user:
               raise HTTPException(status_code=401, detail="用户不存在")
           return user
       except JWTError:
           raise HTTPException(status_code=401, detail="Token无效或已过期")

   def require_role(*roles: str):
       """角色权限校验依赖"""
       async def checker(current_user = Depends(get_current_user)):
           if current_user.role not in roles:
               raise HTTPException(status_code=403, detail="权限不足")
           return current_user
       return checker
```

```
            for attempt in range(3):
```

## 四、接口规范（RESTful）

### 统一URL风格

```
GET    /api/students          → 列表查询（query参数传筛选条件）
POST   /api/students          → 新增（body传JSON）
PUT    /api/students/{id}     → 编辑（body传JSON）
DELETE /api/students/{id}     → 删除单个
POST   /api/students/batch-delete  → 批量删除（body传 { ids: [...] }）
POST   /api/students/import        → 批量导入（multipart/form-data上传文件）
GET    /api/students/export        → 批量导出（返回文件流）
```

### 统一响应格式（所有接口）

```json
// 成功
{ "code": 200, "msg": "操作成功", "data": { "items": [...], "total": 100 } }

// 失败
{ "code": 400, "msg": "班级ID不存在", "data": null }

// 权限不足
{ "code": 403, "msg": "权限不足", "data": null }

// Token无效
{ "code": 401, "msg": "Token无效或已过期", "data": null }
```

### 鉴权方式

```
所有需要鉴权的接口在请求头携带：
Authorization: Bearer <jwt_token>

路由层通过 Depends(get_current_user) 自动校验
需要特定角色的路由通过 Depends(require_role("admin", "teacher")) 校验
```

## 六、安全红线（对照检查清单）

| 检查项 | 正确做法 | 禁止 |
|---|---|---|
| 密码存储 | `passlib[bcrypt]` 哈希 | 明文或MD5 |
| SQL查询 | SQLAlchemy ORM参数化查询 | `f"SELECT ... WHERE id={user_id}"` |
| 用户输入 | Pydantic Schema校验类型和格式 | 直接用request.get('input') |
| API密钥 | `os.getenv()` / pydantic-settings读.env | 硬编码 `api_key = 'sk-xxx'` |
| 文件上传 | uuid重命名 + 限制扩展名(.xlsx/.xls) + 限制大小 | 直接使用用户文件名 |
| 错误信息 | `return {"code": 500, "msg": "服务器内部错误"}` | `return str(e)` 暴露堆栈 |

## 八、与其他工程师的协作边界

### 与数据库工程师
- 你依赖数据库工程师建好MySQL数据库 `aistu`
- ORM模型定义在 `database/models/`，你负责写，数据库工程师负责评审
- 任何表结构变更通过Alembic迁移脚本同步

### 与前端工程师
- 接口文档地址：`http://localhost:8000/docs`（FastAPI自动生成）
- 前后端通过 `api/` 层对接，不要私下修改接口参数格式
- 如果需要调整接口，先沟通，同步修改前后端
