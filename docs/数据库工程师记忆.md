## 二、核心设计原则

### 无外键约束

所有表之间的关联**仅通过ID字段实现，不设置FOREIGN KEY约束**。

理由：
- 数据量较小（中小学校园场景），不需要外键性能优化
- 删除操作更灵活，由业务层控制级联逻辑
- 避免外键锁导致的性能问题
- 批量导入数据时不受外键校验阻塞

### 数据一致性由后端业务层保证

由于无外键，以下检查必须由后端 `services/` 层实现：

| 操作 | 后端必须检查 |
|---|---|
| 删除学生 | 是否存在关联成绩？有则拒绝 |
| 删除教师 | 是否为某班班主任？是则拒绝 |
| 删除班级 | 班级内是否有学生/成绩？有则拒绝 |
| 新增/编辑学生 | class_id是否真实存在？班级是否已满？ |
| 批量导入 | 逐行校验关联ID有效性，返回错误明细 |

## 四、ID关联关系

```
user (用户)
 ├── 1:N → student (student.class_id → class.id)
 ├── 1:N → score (score.student_id → student.id, score.class_id → class.id)
 ├── 1:N → ai_history (ai_history.user_id → user.id)
 └── 1:N → school_rule (school_rule.updated_by → user.id)

class (班级)
 ├── N:1 → teacher (class.head_teacher_id → teacher.id)
 └── 1:N → student (student.class_id → class.id)

teacher (教师)
 └── 1:N → class (teacher.class_ids 逗号分隔 → class.id)
```

## 五、索引设计

| 表 | 索引名 | 类型 | 用途 |

| score | idx_class_id | 普通 | 按班级查成绩 |
| score | idx_exam_batch | 普通 | 按考试批次查成绩 |

| ai_history | idx_tool_type | 普通 | 按功能类型筛选 |
| school_rule | idx_category | 普通 | 按校规分类筛选 |

score表超50万条时，考虑添加联合索引 `(student_id, exam_batch)`。

## 六、特殊字段规则

### teacher.class_ids（逗号分隔）

```
示例值："1,3,5" → 该教师绑定班级ID为1、3、5
```

后端处理：Python拆分后传给SQLAlchemy `.in_()` 查询。
局限：班级绑定数>20时需改为关联表 `teacher_class(teacher_id, class_id)`。

### class.current_count（业务层维护）

| 操作 | current_count 变化 |
|---|---|
| 学生新增/设置class_id | 目标班级 +1 |
| 学生修改class_id | 原班级 -1，新班级 +1 |
| 学生删除 | 所在班级 -1 |
| 批量关联 | 班级 +N |
| 批量移除 | 班级 -N |
| 校务分班 | 涉及班级重新计算 |

## 十、ORM对齐检查清单

后端工程师在 `database/models/` 定义的SQLAlchemy模型必须与你的DDL完全对齐：

- [ ] 表名一致
- [ ] 字段名一致（Python snake_case）
- [ ] 字段类型对齐（BigInteger↔BIGINT, String(50)↔VARCHAR(50), Text↔TEXT）
- [ ] 每个表都有主键 `id`
- [ ] created_at + updated_at 时间戳
- [ ] 索引通过 `__table_args__` 定义
- [ ] **无外键**（不使用 `ForeignKey()`）
- [ ] 字符集 utf8mb4

## 十一、协作边界

### 与后端工程师

- 你负责：建库建表、索引优化、初始化数据、DDL文档
- 后端负责：ORM模型编写（对齐DDL）、Alembic迁移脚本
- 表结构变更流程：你修改DDL → 后端同步修改ORM → 生成Alembic迁移 → 执行
- ORM模型在 `backend/database/models/` 下，后端写但你负责评审

### 密码方案备注

需求文档提到MD5，但架构决策采用bcrypt。user表的password_hash字段存储bcrypt哈希值（60字符），不是32字符的MD5。初始管理员密码需通过应用生成bcrypt哈希后插入。
