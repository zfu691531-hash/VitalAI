## 二、核心设计原则

### 1. 无外键约束

所有表之间的关联**仅通过ID字段实现，不设置FOREIGN KEY约束**。原因：

- 数据量较小（中小学校园场景），不需要外键性能优化
- 删除操作更灵活，由业务层控制级联逻辑
- 避免外键锁导致的性能问题
- 批量导入数据时不受外键校验阻塞

### 2. 数据一致性由业务层保证

由于无外键，以下一致性检查必须由后端 `services/` 层在代码中实现：

| 操作 | 业务层必须检查的内容 |
|---|---|
| 删除学生 | 是否存在关联成绩？ → 有则拒绝删除并提示 |
| 删除教师 | 是否为某班班主任？ → 是则拒绝删除并提示"请先更换班主任" |
| 删除班级 | 班级内是否有学生？是否有学生成绩？ → 有则拒绝删除 |
| 删除成绩 | 无需特殊检查（成绩是末端数据） |
| 关联操作 | 学生绑定班级时，class_id是否真实存在？ |
| 批量导入 | 逐行校验关联ID的有效性，返回错误明细 |

### 3. ID关联关系图

```
user (用户)
 │
 ├── 1:1 ──→ student (学生信息, 通过user关联，或student表独立user_id)
 │                 │
 │                 └── N:1 ──→ class (班级, 通过 student.class_id)
 │                              │
 │                              ├── N:1 ──→ teacher (班主任, 通过 class.head_teacher_id)
 │                              │             │
 │                              │             └── 1:N ──→ class (绑定班级, 通过 teacher.class_ids)
 │                              │
 │                              └── 1:N ──→ student (班级内学生)
 │
 ├── 1:N ──→ score (成绩, 通过 score.student_id + score.class_id)
 │
 ├── 1:N ──→ ai_history (AI历史, 通过 ai_history.user_id)
 │
 └── 1:N ──→ school_rule (校规, 通过 school_rule.updated_by)

备注：student表与user表的关系 — 用户登录后根据role判断，如果是student角色则关联student表
```

## 三、数据库建表DDL

以下为7张表的完整DDL，可直接在MySQL中执行。后端ORM模型必须与此完全对齐。

### 1. 用户表（user）

```sql
CREATE TABLE `user` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `username`    VARCHAR(50)  NOT NULL                COMMENT '登录账号，唯一',
    `password_hash` VARCHAR(255) NOT NULL              COMMENT '密码哈希(bcrypt)',
    `role`        ENUM('student','teacher','admin') NOT NULL COMMENT '角色',
    `name`        VARCHAR(50)  NOT NULL                COMMENT '真实姓名',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2. 学生表（student）

```sql
CREATE TABLE `student` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `name`        VARCHAR(50)  NOT NULL                COMMENT '姓名',
    `gender`      ENUM('male','female') NOT NULL       COMMENT '性别',
    `age`         INT          DEFAULT NULL             COMMENT '年龄',
    `class_id`    BIGINT       DEFAULT NULL             COMMENT '所属班级ID（关联class.id，无外键）',
    `contact`     VARCHAR(20)  DEFAULT NULL             COMMENT '联系方式',
    `specialty`   VARCHAR(200) DEFAULT NULL             COMMENT '个人特长',
    `tags`        VARCHAR(500) DEFAULT NULL             COMMENT '标签，逗号分隔（如：留守儿童,学困生）',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_class_id` (`class_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='学生表';
```

### 3. 教师表（teacher）

```sql
CREATE TABLE `teacher` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `name`        VARCHAR(50)  NOT NULL                COMMENT '姓名',
    `subject`     VARCHAR(50)  DEFAULT NULL             COMMENT '任教科目',
    `title`       ENUM('head_teacher','normal') NOT NULL DEFAULT 'normal' COMMENT '职务：班主任/普通教师',
    `class_ids`   VARCHAR(500) DEFAULT NULL             COMMENT '绑定班级ID列表，逗号分隔（关联class.id，无外键）',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='教师表';
```

### 4. 班级表（class）

```sql
CREATE TABLE `class` (
    `id`                BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `grade`             VARCHAR(20)  NOT NULL                COMMENT '年级（如：2024级）',
    `name`              VARCHAR(50)  NOT NULL                COMMENT '班级名称，唯一（如：2024级1班）',
    `head_teacher_id`   BIGINT       DEFAULT NULL             COMMENT '班主任ID（关联teacher.id，无外键）',
    `max_count`         INT          NOT NULL DEFAULT 50      COMMENT '人数上限',
    `current_count`     INT          NOT NULL DEFAULT 0       COMMENT '当前人数',
    `created_at`        DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`        DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_name` (`name`),
    KEY `idx_grade` (`grade`),
    KEY `idx_head_teacher_id` (`head_teacher_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='班级表';
```

### 5. 成绩表（score）

```sql
CREATE TABLE `score` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `student_id`  BIGINT       NOT NULL                COMMENT '学生ID（关联student.id，无外键）',
    `class_id`    BIGINT       NOT NULL                COMMENT '班级ID（关联class.id，无外键）',
    `exam_batch`  VARCHAR(50)  NOT NULL                COMMENT '考试批次（如期中、期末）',
    `subject`     VARCHAR(50)  NOT NULL                COMMENT '科目',
    `score`       DECIMAL(5,2) NOT NULL                COMMENT '分数（0-100）',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_student_id` (`student_id`),
    KEY `idx_class_id` (`class_id`),
    KEY `idx_exam_batch` (`exam_batch`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成绩表';
```

### 6. AI历史记录表（ai_history）

```sql
CREATE TABLE `ai_history` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `user_id`     BIGINT       NOT NULL                COMMENT '用户ID（关联user.id，无外键）',
    `tool_type`   VARCHAR(50)  NOT NULL                COMMENT '功能类型（comment/discipline/notice/rule_qa/score_diag/meeting/interview/group）',
    `content`     TEXT         NOT NULL                COMMENT 'AI生成的内容',
    `student_id`  BIGINT       DEFAULT NULL             COMMENT '关联学生ID（可选）',
    `class_id`    BIGINT       DEFAULT NULL             COMMENT '关联班级ID（可选）',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_tool_type` (`tool_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI生成历史记录表';
```

### 7. 校规表（school_rule）

```sql
CREATE TABLE `school_rule` (
    `id`          BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `title`       VARCHAR(200) NOT NULL                COMMENT '校规标题',
    `content`     TEXT         NOT NULL                COMMENT '校规正文',
    `updated_by`  BIGINT       DEFAULT NULL             COMMENT '最后更新人ID（关联user.id，无外键）',
    `created_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_at`  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='校规表';
```

## 六、teacher.class_ids 存储规则

教师可绑定多个班级，使用逗号分隔存储在 `class_ids` 字段中：

```
示例值："1,3,5"  →  该教师绑定班级ID为1、3、5的三个班
```

**后端业务层处理要点**：
- 查询时：`WHERE class_id IN (split(teacher.class_ids, ','))` 或在Python中拆分后传给SQLAlchemy
- 绑定时：去重后拼接为逗号分隔字符串
- 解绑时：从字符串中移除指定ID

**注意**：这是用字符串存数组的设计妥协。如果后期数据量大（班级数>20），考虑改为关联表 `teacher_class(teacher_id, class_id)`。

## 七、class.current_count 维护规则

`current_count`（当前人数）字段由后端业务层在以下操作时同步更新：

| 操作 | current_count 变化 |
|---|---|
| 学生新增/编辑（设置class_id） | 目标班级 +1 |
| 学生编辑（修改class_id） | 原班级 -1，新班级 +1 |
| 学生删除 | 所在班级 -1 |
| 批量关联学生到班级 | 班级 +N |
| 批量移除班级学生 | 班级 -N |
| 校务分班 | 涉及的所有班级重新计算 |

## 九、ORM模型与DDL对齐检查清单

后端工程师在 `backend/database/models/` 中定义的SQLAlchemy模型，**必须与本DDL完全对齐**：

| 检查项 | 要求 |
|---|---|
| 表名 | 与DDL的表名完全一致 |
| 字段名 | 与DDL的列名完全一致（Python用snake_case） |
| 字段类型 | BigInteger ↔ BIGINT, String(50) ↔ VARCHAR(50), Text ↔ TEXT 等 |
| 主键 | 每个表都有 `id = mapped_column(BigInteger, primary_key=True)` |
| 时间戳 | `created_at` + `updated_at`，使用 `func.now()` 默认值 |
| 索引 | 通过 `__table_args__` 定义与DDL一致的索引 |
| **无外键** | 不使用 `ForeignKey()`，仅用普通 `BigInteger` 字段存储ID |
| 字符集 | 表级别和连接级别均为 `utf8mb4` |

## 十、与其他工程师的协作边界

### 与后端工程师
- 你负责：建库建表、索引优化、初始化数据
- 后端负责：编写ORM模型（必须与你提供的DDL对齐）、Alembic迁移脚本
- 表结构变更流程：你修改DDL → 后端同步修改ORM模型 → 生成Alembic迁移 → 执行
