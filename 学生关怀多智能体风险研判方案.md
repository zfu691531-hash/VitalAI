#### ⏱️ 时间衰减机制

> 💡 **设计理念**：近期事件权重更高，历史事件影响逐渐衰减，符合教育场景的实际观察规律。

```python
# ============================================================
# 时间衰减窗口配置
# 格式：(天数上限, 衰减因子)
# ============================================================
TIME_DECAY_WINDOWS = (
    (7, 1.0),    # 7天内：100%权重 - 近期事件影响最大
    (30, 0.75),  # 30天内：75%权重 - 逐渐衰减
    (90, 0.45),  # 90天内：45%权重 - 历史事件影响减弱
)
# 超过90天：20%权重（兜底值）

def _apply_time_decay(weight: float, occurred_at: date) -> float:
    """
    应用时间衰减因子到信号权重
    
    Args:
        weight: 信号的原始权重值
        occurred_at: 事件发生日期
    
    Returns:
        float: 应用衰减后的最终权重
    
    Example:
        >>> _apply_time_decay(0.4, date(2024, 1, 1))  # 假设今天是2024-1-20
        0.3  # 20天前，应用75%衰减
    """
    # 计算事件距今的天数
    days = (date.today() - occurred_at).days
    
    # 遍历衰减窗口，找到匹配的衰减因子
    for window, factor in TIME_DECAY_WINDOWS:
        if days <= window:
            return weight * factor  # 返回衰减后的权重
    
    # 超过所有窗口，使用兜底衰减因子20%
    return weight * 0.2
```

**衰减效果示例：**

| 事件类型 | 发生时间 | 原始权重 | 衰减后权重 |
|----------|----------|----------|------------|
| 冲突事件 | 3天前 | 0.40 | 0.40 (100%) |
| 冲突事件 | 20天前 | 0.40 | 0.30 (75%) |
| 冲突事件 | 60天前 | 0.40 | 0.18 (45%) |
| 冲突事件 | 120天前 | 0.40 | 0.08 (20%) |

#### 📝 文本极性分类

> 💡 **设计理念**：自动识别文本描述中的正负情感倾向，区分风险信号与保护性信号。

```python
# ============================================================
# 文本极性分类关键词库
# 用于自动识别文本描述中的正负情感倾向
# ============================================================

# 负向关键词：出现这些词倾向于判断为风险信号
TEXT_NEGATIVE_HINTS = (
    "低落", "焦虑", "孤立", "冲突", "打骂", "欺凌", "受伤",
    "害怕", "不耐烦", "紧张", "回避", "独处", "迟到", "缺勤",
    "早退", "请假", "困难", "压力", "异常", "波动", "无助",
    "失控", "退缩",
)

# 正向关键词：出现这些词倾向于判断为保护性信号
TEXT_POSITIVE_HINTS = (
    "好转", "缓解", "稳定", "支持", "陪伴", "参与", "积极",
    "主动", "融入", "适应", "愿意", "恢复", "配合", "改善",
    "鼓励", "同伴支持", "共同活动", "家长支持", "情绪稳定",
)


def _classify_text_polarity(text: str) -> str:
    """
    基于关键词匹配对文本进行极性分类
    
    Args:
        text: 待分类的文本内容
    
    Returns:
        str: 极性分类结果
            - "negative": 负向，表示风险信号
            - "positive": 正向，表示保护性信号
            - "neutral": 中性，无明显倾向
    
    Example:
        >>> _classify_text_polarity("情绪低落，不愿参与集体活动")
        'negative'
        >>> _classify_text_polarity("情绪稳定，主动融入班级")
        'positive'
    """
    # 统计负向关键词命中次数
    negative_hits = sum(1 for kw in TEXT_NEGATIVE_HINTS if kw in text)
    
    # 统计正向关键词命中次数
    positive_hits = sum(1 for kw in TEXT_POSITIVE_HINTS if kw in text)
    
    # 比较正负命中数，返回极性分类结果
    if negative_hits > positive_hits and negative_hits > 0:
        return "negative"  # 负向词更多，判定为风险信号
    if positive_hits > negative_hits and positive_hits > 0:
        return "positive"  # 正向词更多，判定为保护性信号
    return "neutral"  # 无明显倾向，判定为中性
```

**分类应用场景：**

| 数据源 | 文本示例 | 极性判定 |
|--------|----------|----------|
| 关怀观察 | "情绪低落，不愿参与集体活动" | negative |
| 关怀观察 | "情绪稳定，主动融入班级" | positive |
| 家校沟通 | "家长配合，愿意共同关注" | positive |
| 家校沟通 | "家长不耐烦，沟通困难" | negative |

#### 🛡️ 保护性证据机制

> 💡 **设计理念**：正向信号可产生负权重，主动降低风险分数，体现"保护性因素"的价值。

```python
def _polarity_weight(
    base_weight: float,
    polarity: str,
    positive_factor: float = 0.6,
    neutral_factor: float = 0.0
) -> float:
    """
    根据文本极性调整信号权重
    
    核心设计理念：
    - 负向信号：保持原权重，增加风险分数
    - 正向信号：产生负权重，主动降低风险分数（保护性证据）
    - 中性信号：接近零权重，不显著影响风险分数
    
    Args:
        base_weight: 基础权重值（正数）
        polarity: 文本极性，可选值：'negative', 'positive', 'neutral'
        positive_factor: 正向信号的降权因子，默认0.6
                        即正向信号产生 -base_weight * 0.6 的负权重
        neutral_factor: 中性信号的权重因子，默认0.0
                        即中性信号权重接近0
    
    Returns:
        float: 调整后的最终权重
            - 正数：增加风险分数
            - 负数：降低风险分数（保护性证据）
            - 零：不影响风险分数
    
    Example:
        >>> _polarity_weight(0.35, "negative")
        0.35   # 负向信号，增加风险
        >>> _polarity_weight(0.35, "positive")
        -0.21  # 正向信号，降低风险（保护性证据）
        >>> _polarity_weight(0.35, "neutral")
        0.0    # 中性信号，不影响风险
    """
    if polarity == "negative":
        # 负向：保持原权重（增加风险）
        return base_weight
    if polarity == "positive":
        # 正向：产生负权重（降低风险），体现保护性因素价值
        return -abs(base_weight) * positive_factor
    # 中性：接近零权重，影响极小
    return base_weight * neutral_factor
```

**保护性证据效果：**

| 观察类型 | 文本内容 | 基础权重 | 极性 | 最终权重 |
|----------|----------|----------|------|----------|
| 关怀谈话 | "情绪低落，回避交流" | 0.35 | negative | **+0.35** |
| 关怀谈话 | "情绪好转，愿意参与活动" | 0.35 | positive | **-0.21** |
| 家校沟通 | "家长支持，配合关注" | 0.25 | positive | **-0.19** |
| 家校沟通 | "家长不耐烦，沟通困难" | 0.25 | negative | **+0.25** |

> 🎯 **关键意义**：保护性证据机制让系统能够识别"好转"、"支持"等正向信号，降低整体风险分数，而非只能累加风险。

#### 📊 缺失数据机制

> 💡 **设计理念**：当某类数据缺失时，生成权重为0的"数据缺口信号"，提示用户证据充分度有限，但不会错误地增加或降低风险分数。

```python
def _append_data_gap_signal(
    student: Student,
    signals_to_create: list,
    dimension: str,
    signal_type: str,
    signal_text: str,
) -> None:
    """
    添加数据缺口信号到信号列表
    
    设计理念：
    当某类数据缺失时，生成权重为0的信号，提示用户证据充分度有限，
    但不会错误地增加或降低风险分数。
    
    Args:
        student: 学生对象
        signals_to_create: 信号列表（会被原地修改）
        dimension: 所属维度，如 'emotion', 'study', 'behavior'
        signal_type: 信号类型标识，如 'score_missing', 'attendance_missing'
        signal_text: 信号描述文本，用于向用户说明缺失情况
    
    Returns:
        None: 函数通过原地修改 signals_to_create 列表生效
    
    Example:
        >>> signals = []
        >>> _append_data_gap_signal(
        ...     student,
        ...     signals,
        ...     "study",
        ...     "score_missing",
        ...     "当前缺少成绩记录，建议结合学习状态进一步观察"
        ... )
        >>> signals[0].signal_weight
        0.0  # 权重为0，不影响风险分数计算
    """
    signals_to_create.append(
        StudentCareSignal(
            student_id=student.id,      # 关联学生ID
            signal_type=signal_type,    # 信号类型标识
            dimension=dimension,        # 所属维度
            signal_text=signal_text,    # 用户可见的描述文本
            signal_weight=0.0,          # 权重为0，不影响分数计算
            source="data_gap",          # 标记为数据缺口来源
        )
    )
```

**缺失数据场景：**

| 缺失类型 | 维度 | 提示文本 |
|----------|------|----------|
| 成绩缺失 | study | "当前缺少成绩记录，建议结合学习状态进一步观察" |
| 出勤缺失 | behavior | "当前缺少出勤记录，暂不额外提升行为风险" |
| 关怀观察缺失 | emotion | "当前缺少关怀观察记录，情绪与社交判断的证据充分度有限" |
| 家校沟通缺失 | family | "当前缺少家校沟通记录，家庭支持判断的证据充分度有限" |
| AI摘要缺失 | emotion | "当前缺少AI助手摘要，文本线索型证据暂不充分" |

**数据质量评估：**

```python
def _build_data_quality_summary(signals: list) -> dict:
    """
    构建数据质量评估摘要
    
    用于评估当前学生画像的证据充分度，帮助用户了解：
    - 哪些数据源缺失
    - 有多少正向/保护性信号
    - 整体证据是否足够支撑风险判断
    
    Args:
        signals: 学生关怀信号列表
    
    Returns:
        dict: 数据质量评估结果，包含以下字段：
            - missing_sources: 缺失的数据源列表
            - missing_count: 缺失数据源数量
            - positive_signal_count: 正向信号数量（增加风险）
            - protective_signal_count: 保护性信号数量（降低风险）
            - evidence_sufficient: 证据是否充分（布尔值）
    
    Example:
        >>> _build_data_quality_summary(signals)
        {
            'missing_sources': [...],      # 缺失的数据源
            'missing_count': 2,            # 缺失2个数据源
            'positive_signal_count': 5,    # 5个正向信号
            'protective_signal_count': 2,  # 2个保护性信号
            'evidence_sufficient': False   # 证据不充分（缺失>1个）
        }
    """
    # 筛选数据缺口信号（source="data_gap"）
    missing_sources = [s for s in signals if s.source == "data_gap"]
    
    # 筛选正向信号（权重>0，增加风险）
    positive_signals = [s for s in signals if s.signal_weight > 0]
    
    # 筛选保护性信号（权重<0，降低风险）
    protective_signals = [s for s in signals if s.signal_weight < 0]
    
    return {
        # 缺失数据源详情
        "missing_sources": missing_sources,
        # 缺失数据源数量
        "missing_count": len(missing_sources),
        # 正向信号数量
        "positive_signal_count": len(positive_signals),
        # 保护性信号数量
        "protective_signal_count": len(protective_signals),
        # 证据充分性判断：缺失≤1个 且 正向信号≥2个
        "evidence_sufficient": len(missing_sources) <= 1 and len(positive_signals) >= 2,
    }
```

> 🎯 **关键意义**：系统不会因数据缺失而误判风险，同时会明确告知用户当前证据的充分程度。

## 🏗 技术架构

### 多智能体协同架构

采用 **LangGraph StateGraph** 实现 `6 个维度专家智能体（并行） + 1 个整合智能体` 的工作流：

```
START
  ├─ emotion_agent (情绪专家) ──┐
  ├─ social_agent (社交专家)  ──┤
  ├─ safety_agent (安全专家)  ──┤
  ├─ family_agent (家庭专家)  ──┼──> integration_agent ──> END
  ├─ study_agent (学习专家)   ──┤
  └─ behavior_agent (行为专家)──┘
```

**核心代码结构：**

```python
# ============================================================
# 多智能体工作流状态定义
# 使用 TypedDict 定义状态结构，total=False 表示所有字段可选
# ============================================================
class StudentCareGraphState(TypedDict, total=False):
    """
    LangGraph 工作流状态定义
    
    状态在工作流节点间传递和更新：
    - START → 6个专家节点（并行） → 整合节点 → END
    """
    prompt_payload: dict        # 提示词负载，包含学生基本信息
    profile: dict               # 学生画像数据（六维分数、风险等级等）
    signals: list[dict]         # 风险信号列表，用于支撑研判
    expert_outputs: Annotated[list[dict], operator.add]  # 并行结果汇聚
                                # Annotated + operator.add 实现自动合并
    result: StudentCareAgentResult | None  # 最终研判结果


# ============================================================
# 工作流构建函数
# 使用 @lru_cache 缓存，确保只构建一次
# ============================================================
@lru_cache(maxsize=1)
def _build_student_care_agent_graph():
    """
    构建多智能体研判工作流
    
    工作流结构：
    START
      ├─ emotion_agent (情绪专家) ──┐
      ├─ social_agent (社交专家)  ──┤
      ├─ safety_agent (安全专家)  ──┤
      ├─ family_agent (家庭专家)  ──┼──> integration_agent ──> END
      ├─ study_agent (学习专家)   ──┤
      └─ behavior_agent (行为专家)──┘
    
    Returns:
        StateGraph: 编译后的工作流图
    """
    # 创建状态图，指定状态类型
    workflow = StateGraph(StudentCareGraphState)
    
    # 添加6个维度专家节点（并行执行）
    # 每个专家独立分析一个维度，互不依赖
    for dimension in ["emotion", "social", "safety", "family", "study", "behavior"]:
        node_name = f"{dimension}_agent"
        # 添加节点：节点名称 + 节点处理函数
        workflow.add_node(node_name, _build_dimension_expert_node(dimension))
        # 添加边：START → 专家节点（并行启动）
        workflow.add_edge(START, node_name)
    
    # 添加整合节点：汇聚6个专家结果，生成最终研判
    workflow.add_node("integration_agent", _integration_agent_node)
    # 添加边：所有专家节点 → 整合节点
    workflow.add_edge(expert_node_names, "integration_agent")
    # 添加边：整合节点 → END
    workflow.add_edge("integration_agent", END)
    
    # 编译工作流，返回可执行图
    return workflow.compile(name="student_care_multi_agent")
```

**核心优势：**

| 优势 | 说明 |
|------|------|
| ✅ 并行执行 | 6个专家同时调用大模型，总耗时≈单个专家耗时 |
| ✅ 状态汇聚 | `Annotated[list[dict], operator.add]` 自动汇聚并行结果 |
| ✅ 节点级容错 | 每个专家独立捕获异常，失败不影响其他专家 |
| ✅ 兜底机制 | 模型超时或JSON解析失败时自动使用规则引擎结果 |

## ⚙️ 核心特性

### 心理维度保守判定

```python
# ============================================================
# 心理维度证据来源白名单
# 只有来自以下数据源的信号才能作为心理风险证据
# ============================================================
ALLOWED_EMOTION_SOURCES = {
    "score",             # 成绩数据：成绩下滑可能反映情绪问题
    "attendance",        # 出勤数据：请假备注可能提及情绪
    "behavior_event",    # 行为事件：冲突、违纪等
    "family_contact",    # 家校沟通：家长反馈的情绪问题
    "assistant_summary", # AI助手对话：学生主动披露情绪困扰
    "care_observation",  # 关怀观察：教师的直接观察记录
}
# 注意：标签（tag）不在此白名单中
# 原因：单纯被打标签不能作为心理风险证据，需要更多事实支撑


def _validate_emotion_result(result, signals, tag_definitions):
    """
    验证心理维度研判结果的有效性
    
    设计理念：
    心理维度是最敏感的维度，需要更严格的验证规则：
    1. 高分需要更多证据支撑
    2. 不能仅依赖标签判定
    3. 不能使用正向/中性标签作为风险证据
    
    Args:
        result: 研判结果，包含分数和分析
        signals: 信号列表
        tag_definitions: 标签定义映射
    
    Returns:
        tuple: (是否有效, 错误信息)
            - (True, "") - 验证通过
            - (False, "错误信息") - 验证失败
    """
    # 提取分数和有效证据命中数
    score = result.get("score", 0)
    hits = len([s for s in signals if s.source in ALLOWED_EMOTION_SOURCES])
    
    # 规则1：高分需要更多证据支撑
    # 风险分数>=0.5（中风险）需要至少2个有效证据
    if score >= 0.5 and hits < 2:
        return False, "心理维度高分但证据不足"
    
    # 规则2：关注阈值需要至少1个有效证据
    # 风险分数>=0.3（关注级别）需要至少1个有效证据
    if score >= 0.3 and hits < 1:
        return False, "心理维度评分超过关注阈值但无有效证据"
    
    # 规则3：不能使用正向/中性标签作为风险证据
    # 防止"心理关爱"标签被误判为风险证据
    if _contains_positive_or_neutral_tag(result.get("evidence", []), tag_definitions):
        return False, "心理维度证据包含正向/中性标签"
    
    return True, ""
```

### 核心公式

```python
def _calculate_posterior(prior: float, likelihood_ratios: list[float]) -> float:
    """
    基于似然比累乘计算后验概率（Odds-LR方法）
    
    贝叶斯定理变形：
    P(Risk|Evidence) = P(Evidence|Risk) * P(Risk) / P(Evidence)
    
    使用赔率（Odds）形式简化计算：
    Odds' = Odds × LR1 × LR2 × ... × LRn
    
    Args:
        prior: 先验概率 P(Risk)，范围 [0, 1]
        likelihood_ratios: 似然比列表 [LR1, LR2, ...]
            LR > 1: 增加风险概率（风险证据）
            LR < 1: 降低风险概率（保护性证据）
            LR = 1: 不影响概率
    
    Returns:
        float: 后验概率 P(Risk|Evidence)，范围 [0, 1]
    
    Example:
        >>> _calculate_posterior(0.15, [2.0, 1.5, 0.8])
        # Odds = 0.15 / 0.85 = 0.176
        # Odds' = 0.176 × 2.0 × 1.5 × 0.8 = 0.424
        # Posterior = 0.424 / 1.424 = 0.298
        0.298
    """
    # Step 1: 先验概率转赔率
    # Odds = P / (1 - P)
    odds = prior / (1 - prior)
    
    # Step 2: 累乘似然比
    # Odds' = Odds × LR1 × LR2 × ... × LRn
    # 使用 max(lr, 0.01) 防止似然比过小导致数值问题
    for lr in likelihood_ratios:
        odds *= max(lr, 0.01)
    
    # Step 3: 赔率转回概率
    # P = Odds / (1 + Odds)
    return odds / (1 + odds)


# ============================================================
# 混合公式：线性评分与贝叶斯后验的加权融合
# ============================================================
def calculate_final_score(
    linear_score: float,      # 线性评分层输出的分数
    posterior: float,         # 贝叶斯辅助层输出的后验概率
    blend_alpha: float        # 混合权重参数
) -> float:
    """
    混合线性评分与贝叶斯后验概率
    
    公式：
    FinalScore = α × LinearScore + (1 - α) × Posterior
    
    其中：
    - α (blend_alpha): 线性分权重，通常为 0.7-0.8
    - LinearScore: 基于规则加权求和的分数
    - Posterior: 基于贝叶斯推断的概率
    
    设计理念：
    - 线性评分：反映规则引擎的确定性判断
    - 贝叶斯后验：反映概率视角的风险估计
    - 混合：两者互补，提高鲁棒性
    
    Args:
        linear_score: 线性评分，范围 [0, 1]
        posterior: 贝叶斯后验概率，范围 [0, 1]
        blend_alpha: 线性分权重，范围 [0, 1]
    
    Returns:
        float: 最终风险分数，范围 [0, 1]
    """
    return blend_alpha * linear_score + (1 - blend_alpha) * posterior
```

#### 置信度评估

系统基于证据充分度计算研判置信度：

```python
def _estimate_confidence(signals, root_causes, protective_factors) -> float:
    """
    计算研判置信度
    
    评估维度：
    - 正向信号数量（最多6个，每个+0.05）
    - 图谱信号数量（最多2个，每个+0.05）
    - 信号维度覆盖数（最多4个，每个+0.03）
    - 根因证据数量（最多5个，每个+0.03）
    - 保护性因素（+0.03）
    
    Returns:
        float: 置信度，范围 [0.42, 0.93]
    """
    confidence = 0.42  # 基础置信度
    confidence += min(len(positive_signals), 6) * 0.05
    confidence += min(graph_count, 2) * 0.05
    confidence += min(dimension_count, 4) * 0.03
    confidence += min(root_evidence_count, 5) * 0.03
    if protective_factors:
        confidence += 0.03
    return min(confidence, 0.93)
```

##### 证据充分度评估

系统基于社交数据覆盖情况计算证据充分度：

```python
def _build_social_data_coverage(signals: list[dict], agent_context: dict) -> dict:
    """
    构建社交数据覆盖情况
    
    Returns:
        {
            "covered_count": 3,           # 已覆盖项数
            "required_count": 5,          # 必需项总数
            "coverage_ratio": 0.6,        # 覆盖率
            "evidence_sufficient": True,  # 证据是否充分（覆盖率>=60%）
            "covered_items": [...],       # 已覆盖项列表
            "missing_items": [...]        # 缺失项列表
        }
    """
    covered_items = []
    missing_items = []
    
    for item in SOCIAL_DATA_REQUIREMENTS:
        matched = bool(item["match"](signals, agent_context))
        target = covered_items if matched else missing_items
        target.append({
            "id": item["id"],
            "label": item["label"],
            "description": item["description"]
        })
    
    coverage_ratio = len(covered_items) / len(SOCIAL_DATA_REQUIREMENTS)
    
    return {
        "covered_count": len(covered_items),
        "required_count": len(SOCIAL_DATA_REQUIREMENTS),
        "coverage_ratio": round(coverage_ratio, 4),
        "evidence_sufficient": coverage_ratio >= 0.6,
        "covered_items": covered_items,
        "missing_items": missing_items,
    }
```

def _build_social_trend(signals: list[dict], social_data_coverage: dict) -> dict:

    趋势判定逻辑：

            "recent_support_weight": 0.12, # 近期正向信号权重和

**定位**：解决静态权重、干预策略盲选、教师资源浪费问题

| **干预策略优选** | 把谈心、团建、家校沟通设为「臂」，推荐最优干预方案 |

**接入方式**：仅读取原有研判记录、贝叶斯配置、教师复核状态，输出优化建议，不修改核心逻辑。

**接入方式**：仅读取历史风险分数、信号时序、图谱孤立信号，输出时序状态曲线，叠加展示。

> 本项目面向高中校园学生关怀场景，构建了一套"规则评分 + 双层贝叶斯推断 + 多智能体研判 + 人在回路确认 + 图谱增强"的学生风险研判系统。系统从校内数据中提取风险信号，形成六维关怀画像。在安全、情绪、家庭、社交四个高敏感维度，通过贝叶斯辅助层进行似然比累乘概率修正；针对社交孤立场景，构建了完整的贝叶斯网络（含5个根因节点、9条因果边），实现因果推理和风险传导路径可视化。Neo4j图谱识别冲突共现、社交孤立等关系型风险线索。人在回路机制确保教师对AI研判结果拥有最终裁量权，教师确认结果会作为贝叶斯证据参与下轮概率更新。

### 技术路线版

本项目采用"事实采集层 - 信号抽取层 - 线性评分层 - 双层贝叶斯推断层 - 多智能体解释层 - 人在回路确认层"的分层技术路线：

1. **LangGraph编排6专家并行研判** - 总耗时≈单个专家耗时
2. **双层贝叶斯推断** - 辅助层（似然比累乘）+ 网络层（因果图推理）
3. **教师确认闭环学习** - 确认结果作为贝叶斯证据
4. **Neo4j图谱关系增强** - 冲突共现、社交孤立信号
5. **安全披露自动捕获** - 关键词检测触发画像重算

### 公式化表达

```text
# ============================================================
# 第一层：线性评分层
# 基于规则加权求和，归一化到 [0, 1]
# ============================================================
S_d = Normalize(Σ w_i · x_i),  i ∈ Signals(d)

# 其中：
# S_d: 维度 d 的线性评分
# w_i: 信号 i 的权重（考虑时间衰减）
# x_i: 信号 i 的值（通常为1，表示信号存在）
# Signals(d): 维度 d 的所有信号集合


# ============================================================
# 第二层：贝叶斯辅助层（Odds-LR累乘）
# 使用似然比进行概率修正
# ============================================================
Odds' = Prior/(1-Prior) × ∏ LR_j

# 赔率转概率：
P_d = Odds' / (1 + Odds')

# 其中：
# Prior: 先验概率（如 emotion=0.15）
# LR_j: 第 j 个证据的似然比
# P_d: 后验概率


# ============================================================
# 第三层：贝叶斯网络层（社交孤立场景）
# 使用因果图进行风险传导计算
# ============================================================
P(isolation) = OR_gate([
    P(social_withdrawal) × 0.78,    # 社交退缩传导系数
    P(peer_disconnect) × 0.72,      # 同伴缺失传导系数
    P(safety_threat) × 0.28         # 安全威胁传导系数
])

# OR门函数：取各路径风险的最大值
# 体现"任一风险源触发即可导致孤立"的逻辑


# ============================================================
# 第四层：混合层
# 线性评分与贝叶斯后验的加权融合
# ============================================================
FinalScore_d = λ · S_d + (1 - λ) · P_d

# 其中：
# λ (blend_alpha): 线性分权重，如 0.75
# S_d: 线性评分
# P_d: 贝叶斯后验概率
# FinalScore_d: 最终风险分数
```

### 答辩一句话

> 我们做的不是直接给学生贴标签的黑箱AI，而是基于校内事实信号的可解释学生关怀研判系统。系统采用"线性规则评分 + 贝叶斯概率修正 + 图谱关系增强"混合架构，通过六专家多智能体并行研判和教师确认闭环，实现"模型研判 - 教师复核 - 概率更新"的教育场景闭环管理。人在回路机制确保教师对AI结果拥有最终裁量权，既保证AI辅助价值，又确保教育决策的人类主体性。
