二、核心功能设计
功能1：MAB驱动的智能推送调度——在最对的时间说话
老人最讨厌的是：吃饭时提醒喝水、午睡时提醒运动、看剧时提醒打电话。MAB（多臂老虎机）把一天分为8个时段，通过Thompson Sampling持续学习"王奶奶在哪个时段最愿意响应"，自动把提醒调整到最合适的时间。


功能2：购物比价与协同过滤推荐——不再被复杂的网购迷惑
老年人网购的真实问题不是"不会操作"，而是"不知道哪个好、哪个值"。系统用协同过滤找到"与王奶奶画像相似的老年群体"买什么，再结合价格比较，直接给出最终推荐，而不是把一堆选项扔给老人自己判断。


功能3：出行规划——Dijkstra近途 + PSO远途
近距离出行：改良Dijkstra——老人的"最短路"不是距离最短
对老年人来说，路径规划的优先级是：无台阶 > 路面平整 > 有遮阳 > 距离短。传统Dijkstra算法只考虑距离，但我们需要把"台阶数量"、"路面坡度"、"有无扶手"等因素纳入权重，重新定义老年人的"最优路径"。

远途出行：PSO粒子群优化——复杂多目标行程规划
王奶奶想去一次5天4晚的北京周边游，需要安排：交通（高铁/大巴）、住宿（一楼或有电梯）、景点（步行距离适中）、餐饮（符合饮食限制）。这是一个包含数千种组合的多目标优化问题，PSO可以快速收敛到最优方案。

功能4：防诈骗识别——守护老人的钱包
老年人是电信诈骗的主要受害群体，原因不是老人"蠢"，而是诈骗话术专门针对老年人的心理弱点（权威感、关怀心、恐惧感）设计。系统通过分析通话记录、转账行为、联系人关系三个维度识别风险。

// 老年人专属Dijkstra权重设计
function calcEdgeWeight(segment, user_profile) {
  let weight = segment.distance_meters

  // 台阶惩罚（对步态偏弱的老人，每级台阶=额外走10米）
  const stair_penalty = segment.stairs_count * 10
  // 但如果有扶手，惩罚减半
  const handrail_discount = segment.has_handrail ? 0.5 : 1.0
  weight += stair_penalty * handrail_discount

  // 坡度惩罚（超过5度的坡对老人很危险）
  if (segment.slope_deg > 5) weight += segment.slope_deg * 8

  // 路面质量（石子路、破损路面额外惩罚）
  const surface_penalty = { smooth:1.0, brick:1.3, gravel:2.0, broken:3.0 }
  weight *= surface_penalty[segment.surface] || 1.0

  // 遮阳加分（夏天高温时，有遮阳的路段权重降低）
  const is_summer = isSummer() && isHot()
  if (is_summer && segment.has_shade) weight *= 0.8

  // 根据用户步态状态动态调整（HMM返回cautious时加重台阶惩罚）
  if (user_profile.gait_state === "cautious") weight *= 1.5
  if (user_profile.gait_state === "unsteady") weight *= 2.0

  return weight
}
