二、核心功能设计
功能1：HMM步态异常感知——摔倒前就知道
老年人摔倒前，步态会有明显变化：步长缩短、步频下降、双侧不对称、加速度方差增大。HMM（隐马尔可夫模型）把这些传感器数据建模为隐藏状态序列，从"正常→谨慎→不稳定→跌倒风险"四个状态中实时识别当前位置，并在到达"跌倒风险"之前就预警。



功能2：Fuzzy Logic血压模糊评估——告别一刀切硬阈值
全国统一的血压阈值（收缩压>140=高血压）不适合每个人。王奶奶平时血压偏低（120/75），她的138已经是明显升高；而另一个老人平时145，今天测到148没什么问题。Fuzzy Logic用模糊隶属度函数把血压转化为连续的风险分（0-1），结合个人基线动态校正，比硬阈值准确得多。



功能3：药物冲突检测——用药安全守护
老年人普遍多病共存，同时服用5种以上药物极为常见。药物相互作用复杂，医生可能来自不同医院，相互不知道对方开了什么药。系统在用户录入新药时，自动检查与现有药物的已知冲突，并用自然语言解释风险。


功能4：久坐检测与活动提醒——沉默的健康杀手

功能5：智能体检报告解读
老人拿到体检报告，满页的数字和箭头（↑↓），完全看不懂，子女又不在身边。系统通过OCR识别报告内容，结合个人历史数据，用大白话解释哪些指标正常、哪些需要关注、哪些需要立刻就医，并自动同步到医疗记录数据库。

// HMM步态感知模块
class GaitHMMMonitor {
  // 四个隐藏状态
  STATES = ["normal", "cautious", "unsteady", "fall_risk"]

  // 转移矩阵：从状态i到状态j的概率（基于历史老年步态数据训练）
  TRANSITION = [
    [0.90, 0.08, 0.015, 0.005],  // normal → ...
    [0.15, 0.70, 0.12,  0.03 ],  // cautious → ...
    [0.05, 0.20, 0.60,  0.15 ],  // unsteady → ...
    [0.02, 0.08, 0.30,  0.60 ],  // fall_risk → ...
  ]

  async onNewStep(step_data) {
    const features = {
      stride_len:    calcStrideLength(step_data),   // 步长（cm）
      cadence:       calcCadence(step_data),         // 步频（步/分）
      asymmetry:     calcAsymmetryIndex(step_data),  // 左右不对称0-1
      acc_variance:  calcAccVariance(step_data),     // 加速度方差
      impact_force:  calcImpact(step_data),          // 落脚冲击力
    }
    this.buffer.push(features)

    // 单步加速度突变：不等窗口满，立即分析
    if (features.acc_variance > this.SUDDEN_THRESHOLD) {
      return this.analyzeUrgent()
    }

    // 每20步：滑动窗口分析
    if (this.buffer.length >= 20) {
      await this.analyzeWindow()
      this.buffer = this.buffer.slice(-10)  // 保留后10步继续滑动
    }
  }

  // 个体基线校准（王奶奶的"正常步态"和普通人不同）
  async calibrateBaseline(user_id) {
    const history = await db.getRecentGait(user_id, 30)  // 近30天步态
    this.personal_baseline = {
      avg_stride:    mean(history.map(h=>h.stride_len)),
      avg_cadence:   mean(history.map(h=>h.cadence)),
      normal_variance: stddev(history.map(h=>h.acc_variance)),
    }
    // 王奶奶的"正常步长"是38cm，而非通用标准的50cm
    // 她的38cm不触发告警，但如果降到30cm才是真的异常
  }
}

// 体检报告OCR+解读流程
async function interpretMedicalReport(image_file, user_id) {
  // Step1：OCR识别体检报告
  const ocr_result = await toolAgent.call("ocr_medical", { image: image_file })
  // ocr_result.items = [{name:"血红蛋白",value:105,unit:"g/L",flag:"↓",range:"120-160"}...]

  // Step2：结合个人历史对比
  const history = await db.getMedicalHistory(user_id, 365)  // 过去一年

  // Step3：LLM生成大白话解读
  const interpretation = await llm.generate({
    system: "你是老年人的健康顾问，用亲切的语言解释体检结果，不用专业术语",
    context: { items: ocr_result.items, history, user_baseline: await getBaseline(user_id) },
    instruction: `
      分三部分解读：
      1. 好消息（哪些指标正常或好转）
      2. 需要关注（哪些指标偏高/偏低，原因可能是什么）
      3. 建议行动（是否需要复查、调整用药、改变饮食）
      语言要像朋友聊天，不要让老人担心，但要说清楚`,
  })

  // Step4：自动写入medical_facts表
  await db.updateMedicalHistory(user_id, ocr_result.items)

  return interpretation
}
